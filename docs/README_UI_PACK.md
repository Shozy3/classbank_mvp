# UI Direction Pack

This pack defines the visual and interaction contract for the local macOS MVP.

## Purpose

These files exist to stop implementation drift and keep the UI aligned with the intended product feel:
- premium
- dense but readable
- desktop-oriented
- calm in dark mode
- serious study tool, not a toy

## Recommended implementation order

1. Read `docs/visual-direction.md`
2. Read `docs/design-tokens.md`
3. Read `docs/layout-rules.md`
4. Read `docs/screens/practice-session.md`
5. Build a static prototype of Practice Session first
6. Refine components using `docs/component-rules.md`
7. Then build the rest of the screens

## Most important rule

The Practice Session screen is the design anchor for the whole application.

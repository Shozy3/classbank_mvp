# Visual Direction

## Design intent

The product should feel like a premium, focused study tool for serious technical coursework.

It should feel:
- efficient
- disciplined
- calm
- dense but not cramped
- professional
- desktop-native in spirit even if implemented with cross-platform UI technology later

It should not feel:
- playful
- rounded and bubbly
- mobile-first
- over-animated
- glassy or translucent
- like a generic dashboard template
- like a note-taking app with quizzes stapled onto it

## Primary visual references

### UWorld-inspired qualities to borrow
- split study layout with strong question focus
- efficient use of horizontal space
- immediate review workflow
- persistent navigator
- content-first presentation
- answer/explanation rhythm that supports momentum

### Qualities to avoid copying directly
- exact visual layout duplication
- exact branding patterns
- exact spacing or styling mimicry

Target the study ergonomics, not a clone costume.

## Secondary reference qualities

### Linear-like qualities
- spacing discipline
- restrained surfaces
- premium dark mode
- strong hierarchy without visual clutter

### Raycast-like qualities
- confident density
- sharp focus treatment
- desktop seriousness

### Technical-editor qualities
- code/math/table content should feel native and legible
- syntax-heavy and engineering-heavy material must not feel visually punished

## Visual personality

### Surfaces
Use layered dark surfaces with clear but subtle separation.

Avoid:
- giant floating cards everywhere
- thick borders around every element
- low-contrast mud-gray surfaces
- excessive blur effects

### Density
The UI should be compact and information-efficient.

That means:
- smaller but readable text
- tighter vertical rhythm
- narrow but intentional padding
- meaningful grouping
- minimal dead space

### Tone
The product should say:
“sit down and study”
not
“welcome to my sleek productivity startup.”

## Non-negotiable qualities

1. The practice screen must feel excellent.
2. Explanations must feel readable, not dumped.
3. Dark mode must be calm and high-contrast.
4. Navigation must be obvious and fast.
5. Answer states must be unmistakable.

## Visual anti-goals

Do not use:
- pastel accent palettes
- giant rounded pills everywhere
- glassmorphism
- neumorphism
- oversized spacing
- low-information hero sections
- soft toy-like cards
- generic marketing-site gradients

# Data Model

## Storage approach

Use a local embedded database.

Recommended:
- SQLite for structured data
- images stored as BLOBs in the database for a self-contained MVP

## Rules

### Editing content
- editing a flashcard resets SR history
- editing a short-answer question resets SR history
- editing a standard MCQ may reset or invalidate adaptive review history if content changed materially
- store a revision snapshot on save

### Bookmark vs flag
- bookmark = useful / save for later
- flag = revisit or something is wrong

### Multi-select scoring
- partial credit supported
- full correctness requires all correct options selected and no incorrect options selected

### Session generation
- blocks are generated by filters, not manually assembled

### Topic membership
- a question belongs to one topic only in MVP
- a flashcard belongs to one topic only in MVP
